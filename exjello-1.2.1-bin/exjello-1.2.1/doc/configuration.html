exJello is configured through the properties provided when establishing a JavaMail session.  In particular, the following properties are used:

 Property  Description
org.exjello.mail.mailbox Specifies the Exchange mailbox against which operations will be performed.  This should be the user's e-mail address (e.g., "my.user@example.com"); for older Exchange systems (prior to Exchange 2003 Service Pack 1) this should be set to the address part only (for example, "my.user").  If specified, this will take precedence over the "mail.smtp.from" and "mail.pop3.from" settings.
org.exjello.mail.limit Limits the number of messages that will be retrieved in a single session.  Default is no limit.
org.exjello.mail.unfiltered By default, exJello retrieves only unread messages from the Exchange mailbox.  Specifying "true" here indicates that all messages should be retrieved.
org.exjello.mail.delete By default, messages that are deleted through JavaMail are simply marked as read in Exchange by exJello.  Setting this to "true" indicates that exJello should physically delete the messages from the mailbox instead.
mail.smtp.from
mail.pop3.from		 Specifies the Exchange mailbox against which operations will be performed.  This is used in the same way as "org.exjello.mail.mailbox".
mail.smtp.ssl.enable
mail.pop3.ssl.enable		 Indicates that HTTPS should be used instead of HTTP when connecting to the Outlook Web Access server.
mail.smtp.port
mail.pop3.port		 Specifies the port on the Outlook Web Access server to which exJello should connect.  Defaults to 80 for HTTP and 443 for HTTPS.
mail.smtp.connectiontimeout
mail.pop3.connectiontimeout		 Time in milliseconds that exJello should wait for a connection to be established before throwing an error.
mail.smtp.timeout
mail.pop3.timeout		 Time in milliseconds that exJello should wait for data to be received on the socket before throwing an error.
mail.smtp.localaddress
mail.pop3.localaddress		 The local interface that exJello should use to connect to the remote host.  Useful for multihomed machines.
mail.smtp.user
mail.pop3.user
mail.user		 The username that should be used for making the connection.  This can also be specified directly when obtaining the Store or Transport.  This should typically include the user's domain in the form:

MYDOMAIN\myusername

The username can also include the mailbox information directly by applying a colon-separated suffix; for example:

MYDOMAIN\myusername:my.user@example.com

Additionally, the "limit", "unfiltered", and "delete" options can be included by appending those to the mailbox, enclosed in square brackets.  For example:

MYDOMAIN\myusername:my.user@example.com
[limit=30,unfiltered=true,delete=false]

(the above should be on a single line).  This allows you to embed options in a single configuration setting, for environments where you may not have full control over all JavaMail properties.
mail.smtp.host
mail.pop3.host
mail.host		 Specifies the server to which exJello should connect.  In addition to specifying simply the host name or address, you can also specify a URL to include the protocol and optionally port elements.  For example:

http://server.example.com:8080

This will override the configured port and SSL options to connect to port 8080 on the server over HTTP.  This can be used to embed additional options in a single configuration setting, for environments where you may not have full control over all JavaMail properties.
